## 根因（不变）

`ChatMessageContent.kt` 的 `ChatMessageList`（88-222 行）两个 LaunchedEffect 竞态：
- **Effect A**（114-125）`snapshotFlow` 判断"是否在底部"→ 写 `shouldAutoScroll`
- **Effect B**（127-132）监听 `contentVersion`（含 `streamingPartTexts.hashCode()`）→ `animateScrollToItem`

流式期间每个 token 新建 Map，Effect B 每秒被重启几十次，动画跑不完，列表停中间；Effect A 把 `shouldAutoScroll` 翻 false，停滚，新消息被挤出视口 → 表现"跳到上方"。

辅助因素：`ChatMessageList` 在 Compose 树位置跨 session 不变（`ChatScreen.kt:109-132`），单一 `listState`/`shouldAutoScroll` 跨 session 复用。

## 修复方案（更新：改为记忆每个 session 位置）

只改 `ChatMessageContent.kt` 一个文件。三件事：① 修竞态；② 每个 session 独立 `LazyListState` 实现位置记忆；③ 加"回到底部"按钮。

### 改动清单

**1. 清理 4 个死 import**（78-80, 85 行）
删 `animateContentSize`、`rememberCoroutineScope`、`withFrameNanos`、`launch`。其中 `rememberCoroutineScope`/`launch` 会在第 6 步真实使用时重新引入。

**2. 每 session 独立 `LazyListState`（位置记忆核心）**（替换 101 行）
```kotlin
// 按 sessionId 缓存各自的滚动状态，切 session 位置天然保留
val listStates = remember { mutableStateMapOf<String, LazyListState>() }
val listState = currentSessionId?.let { id ->
    listStates.getOrPut(id) { LazyListState(firstVisibleItemIndex = 0, firstVisibleItemScrollOffset = 0) }
} ?: rememberLazyListState()
val layoutInfo = listState.layoutInfo
```
切走时旧 session 的 listState 留在 map 里（记忆），切回时 `getOrPut` 取回原状态恢复位置。无需改 ViewModel。`mutableStateMapOf` 已在第 54 行 import。

**3. 同步派生"是否在底部"，删除 `shouldAutoScroll` 可变状态**（替换 103 行 + 114-125 行 Effect A）
```kotlin
val atBottom by remember {
    derivedStateOf {
        val info = listState.layoutInfo
        val last = info.visibleItemsInfo.lastOrNull() ?: return@derivedStateOf true
        last.index >= info.totalItemsCount - 1 &&
            last.offset + last.size <= info.viewportSize.height + 24
    }
}
```
`derivedStateOf` 同步读 layoutInfo，自动追踪依赖并防抖（仅布尔翻转才重组）。无 visible items 返回 true（初态贴底）。消除两个 Effect 间状态同步竞态。

**4. 重写滚动执行器，流式用瞬时滚动**（替换 105-111 行 `contentVersion` + 127-132 行 Effect B）
删除 `contentVersion`（竞态根因）。改用 `scrollToItem`（瞬时）替代 `animateScrollToItem`（动画），高频流式下才能稳定贴底：
```kotlin
val bottomAnchor = remember(messages, streamingReasoningPart) {
    messages.size to messages.lastOrNull()?.parts?.size
}
val streamingTrigger = remember(streamingPartTexts) { streamingPartTexts.hashCode() }
LaunchedEffect(atBottom, bottomAnchor) {
    if (atBottom && (messages.isNotEmpty() || streamingReasoningPart != null)) {
        val idx = listState.layoutInfo.totalItemsCount - 1
        if (idx >= 0) listState.scrollToItem(idx)
    }
}
LaunchedEffect(streamingTrigger) {
    if (atBottom && messages.isNotEmpty()) {
        val idx = listState.layoutInfo.totalItemsCount - 1
        if (idx >= 0) listState.scrollToItem(idx)
    }
}
```
两个 effect 都先查 `atBottom`：用户上滑→false→停滚；滑回底部→true→恢复贴底。

**5. 首次进入某 session 滚到底部**（新增，处理"记忆位置"的首次 fallback）
"记忆位置"语义下，首次打开某 session 无位置可记。聊天 app 惯例 fallback 到最新消息。用 per-session 标记区分"首次"与"用户已浏览"：
```kotlin
val initializedSessions = remember { mutableStateMapOf<String, Boolean>() }
LaunchedEffect(currentSessionId, isLoading, messages.isEmpty()) {
    val id = currentSessionId ?: return@LaunchedEffect
    // 仅在该 session 首次加载完成、用户尚未滚动时贴底
    if (!isLoading && messages.isNotEmpty() && initializedSessions[id] != true) {
        val idx = listState.layoutInfo.totalItemsCount - 1
        if (idx >= 0) {
            listState.scrollToItem(idx)
            initializedSessions[id] = true
        }
    }
}
```
首次完成后置 true，之后该 session 完全由各自 listState 记忆位置，不再强制贴底。

**6. "回到底部"浮动按钮**（新增，产品决策 1）
把 `LazyColumn`（167 行）外层套 `Box`，当 `!atBottom && messages.isNotEmpty()` 时右下角显示按钮：
```kotlin
val scope = rememberCoroutineScope()
Box(Modifier.fillMaxSize()) {
    LazyColumn(...) { /* 原内容不变 */ }
    if (!atBottom && messages.isNotEmpty()) {
        Surface(
            onClick = { scope.launch { listState.scrollToItem(listState.layoutInfo.totalItemsCount - 1) } },
            modifier = Modifier.align(Alignment.BottomEnd).padding(16.dp.uiScaled()),
            shape = RoundedCornerShape(20.dp.uiScaled()),
            tonalElevation = 3.dp,
            shadowElevation = 3.dp
        ) { Text("↓ 回到底部", ...) }
    }
}
```
点按钮瞬时滚到底，`atBottom` 随之变 true，按钮消失，恢复自动贴底。重新引入 `rememberCoroutineScope`、`launch`（真实使用，非死 import）。

### 不改动的部分
- ViewModel 层不动（`MainViewModel*.kt`）——根因在 UI 层 Effect 竞态。位置记忆完全在 UI 层用 listState map 实现，与现有草稿记忆模式（`selectSessionState` 160-165 行）解耦。
- 不恢复 `reverseLayout`（commit `07f147e` 去掉它是为修 expand/collapse 跳动 bug）。
- `StreamDebugLogger` 调试 effect（134-150）保留，其 key 里 `shouldAutoScroll` 改为 `atBottom`。

### 边界情况处理
- session 删除：对应 listState 留在 map 占少量内存，MVP 不做主动清理（低频操作）。
- 配置变更（转屏）：`remember` 的 map 不跨进程，但 `LazyListState` 内部有 saveable 机制会保存单实例位置；转屏后 map 重建，各 session 位置从 saveable 恢复。可接受。
- 空 session（currentSessionId == null）：fallback 到普通 `rememberLazyListState()`，不会崩。

### 验证
- `./gradlew assembleDebug` 编译通过
- `./gradlew testDebugUnitTest` 现有单测无回归
- 手动验证：
  1. 发消息触发流式 → 稳定贴底，不上跳
  2. 流式中手动上滑 → 停住浏览；出现"回到底部"按钮
  3. 点按钮 → 瞬间回底，恢复自动贴底
  4. session A 停中间 → 切 B → 切回 A → A 恢复到中间位置（记忆生效）
  5. 首次打开新 session → 自动定位到最新消息

### 风险
- 瞬时 `scrollToItem` 在偶发新消息时比动画"硬"，但仅 `atBottom` 为真时触发，用户主动上滑不受影响。这是聊天列表贴底的标准做法。
- listState map 随打开过的 session 增长，每个 `LazyListState` 内存极小（KB 级），无实际风险。
